# Glossary

Version: 1.0  
Status: Active

---

| Term | Definition |
|------|-------------|
| **Player** | An authenticated account representing an individual user. |
| **Profile** | Public metadata such as name, avatar, and stats for a player. |
| **Inventory** | Collection of player‑owned items including cosmetics and unlockables. |
| **Progression** | XP, level, and XP‑to‑next tracking structure under server control. |
| **Unlock** | Any item, character, or cosmetic rewarded through gameplay milestones. |
| **Quest** | Short‑term objective that grants XP and rewards when completed. |
| **Achievement** | Milestone signifying long‑term progress and associated rewards. |
| **Cloud Save** | Client‑owned but server‑validated state containing preferences and checkpoints. |
| **LiveOps** | Server‑controlled dynamic configurations and temporary events. |
| **Server‑Authoritative** | The server has exclusive authority over any gameplay or economy logic. |
| **Client State** | Ephemeral or non‑authoritative data cached locally on the client. |
| **Repository** | Data persistence layer for database interactions. |
| **Service** | Business logic layer that enforces rules and validation. |
| **API** | HTTP interface exposing validated results and error envelopes. |
| **Domain** | Primary gameplay feature area (Progression, Shop, Achievements, etc.). |
| **Feature Flag** | Configurable key used to enable or disable functionality dynamically. |
| **ADR** | Architecture Decision Record — documents major technical and process choices. |
| **RLS** | Row Level Security — Supabase policy defining access control at row level. |
| **Checksum** | Hash validation ensuring Cloud Save data integrity. |
| **Request ID** | Unique identifier assigned to each API call for traceability. |
| **Constitution** | The overall governance document defining project authority and rules. |
